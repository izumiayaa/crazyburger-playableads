## 1. 文件目的

本規格書描述「瘋狂漢堡」Google Ads HTML5 試玩廣告的完整技術需求與實作規則，供前端工程師依據此文件進行開發、測試及交付。

## 2. 專案概述

### 2.1 廣告形式

- 格式：Google Ads Playable Ad（HTML5，單一自包含 index.html）
- 目標受眾：行動裝置用戶（iOS / Android）
- 主要 CTA：引導用戶前往 Google Play / App Store 下載安裝

### 2.2 遊戲玩法概述

玩家扮演「漢堡大亨」，透過點擊「加料」按鈕不斷疊加漢堡食材，同時累積倍數收益。

- 每次點擊「加料」：新增一層食材，漢堡越來越高
- 特殊食材「洋蔥」：出現時觸發幸運轉盤，隨機抽取爆發倍數
- 點擊「出餐」：結算當前倍數 × 本金，顯示獲得收益
- 漢堡偏移過重：自動倒塌，顯示結算框並可重新遊玩

### 2.3 遊戲主題資訊

廣告以「超級漢堡大亨：疊高高 (Burger Tycoon: Stack High)」作為宣傳主題，評分 4.8 星、5,000 萬+ 下載，強調解壓與休閒體驗。

## 3. 廣告觸發邏輯規格

本廣告共設計三種觸發時機，開發者需依照以下規則實作：

| 觸發條件 | 類型 | 行為描述 | 備註 |
|----------|------|----------|------|
| 失敗重試（每次倒塌後） | 失敗驅動 | 玩家倒塌後再次點擊「加料」時，跳出商店廣告一次，關閉後可繼續遊玩 | 可由開發者設定開/關 |
| 開啟時間 > 12 秒 | 時間驅動 | 玩家點擊「出餐」時，先顯示報獎框，隨後跳出商店廣告 | 報獎後彈窗 |
| 開啟時間 > 25 秒 | 全局攔截 | 需點擊畫面才會觸發廣告彈出；關閉後再次點擊畫面會再次觸發 | 點擊觸發型 |

### 3.1 安全期（Safe Zone）說明

為確保用戶首次體驗廣告時能充分進行互動，系統設計「漢堡安全期」機制：

- 適用範圍：僅限本次廣告展示的「第一局」
- 保護層數：前 4 層擁有完美保護，不會觸發倒塌邏輯
- 解除時機：第 5 層起或非首局時，安全期自動失效
- 再玩一次後：`isFirstSession = false`，後續每局從第 1 層即有倒塌可能性

## 4. 遊戲狀態機 (Game State Machine)

系統以六個主要狀態組成有限狀態機（FSM），各狀態轉換規則如下：

| 狀態名稱 | 觸發條件 | 說明 |
|----------|----------|------|
| IDLE（等待開始） | 廣告首次開啟 | 顯示漢堡底座與「加料」按鈕，等待玩家互動 |
| STACKING（疊加中） | 玩家點擊「加料」 | 每次點擊新增一層食材，洋蔥層觸發倍數機制 |
| ONION_SPIN（洋蔥轉盤） | 食材為洋蔥時 | 顯示洋蔥幸運轉盤動畫，系統隨機抽取爆發倍數 |
| SERVING（出餐） | 玩家點擊「出餐」 | 結算當前倍數 × 本金，顯示獲得收益 |
| FAILED（倒塌） | 漢堡超過物理閾值倒塌 | 顯示結算框（層數、倍數、收益），提供再玩一次按鈕 |
| AD_BREAK（廣告插播） | 符合三種廣告觸發條件之一 | 彈出商店廣告；關閉後恢復上一個遊戲狀態 |

### 4.1 狀態轉換流程

- `IDLE` → `STACKING`：玩家首次點擊「加料」
- `STACKING` → `ONION_SPIN`：食材類型判定為洋蔥
- `ONION_SPIN` → `STACKING`：轉盤結束，倍數更新完成
- `STACKING` → `SERVING`：玩家點擊「出餐」（時間 <= 12 秒）
- `STACKING` → `AD_BREAK` → `SERVING`：玩家點擊「出餐」（時間 > 12 秒）
- `STACKING` → `FAILED`：漢堡倒塌條件成立
- `FAILED` → `AD_BREAK` → `STACKING`：玩家點擊「再玩一次」（`failAdEnabled = true` 且 `adShownOnFail = false`）
- `AD_BREAK` → [前一狀態]：玩家關閉廣告

## 5. UI 元件規格

| 元件 ID | 類型 | 顯示內容 | 互動行為 | 顯示條件 |
|---------|------|----------|----------|----------|
| btn_add | Button | 加料 🍔 | 觸發疊層邏輯；時間 > 25 秒時觸發廣告攔截 | IDLE / STACKING |
| btn_serve | Button | 出餐 $10,000 | 時間 > 12 秒時先彈報獎框再跳廣告；否則直接出餐結算 | STACKING |
| lbl_multiplier | Label | 目前倍數 X.XXx | 即時更新倍數數值 | 常駐 |
| onion_wheel | Overlay | 洋蔥幸運轉盤動畫 | 自動旋轉，結束後顯示倍數結果 | ONION_SPIN |
| result_modal | Modal | 倒塌結算框（層數/倍數/收益） | 「再玩一次」按鈕重置遊戲 | FAILED |
| cta_banner | Banner | 立即安裝遊玩 🚀 | 點擊跳轉 Google Play 安裝頁 | 常駐 / SERVING / FAILED |
| ad_overlay | Overlay | 商店廣告 | 關閉後回到前一個遊戲狀態 | AD_BREAK |
| timer_label | Debug Label | 廣告開啟計時（秒） | 僅開發模式顯示 | Dev Mode |

### 5.1 畫面佈局說明

- 主遊戲區（中央）：漢堡疊加視覺化展示區，採手機比例垂直置中
- 頂部資訊列：倍數顯示 Label、計時器（Dev Mode）
- 底部操作區：「加料」按鈕、「出餐」按鈕
- CTA 橫幅（固定底部）：「立即安裝遊玩 🚀」下載連結
- Overlay 層：洋蔥轉盤、結算 Modal、廣告 Overlay（Z-index 堆疊）

### 5.2 響應式設計

- 手機預覽區：自動偵測裝置尺寸，以 375 × 667（iPhone SE）為基準
- 食材圖示：SVG 或 PNG，支援視網膜螢幕（2x / 3x）
- 字體大小：最小 12pt，確保行動裝置可讀性

## 6. 資料變數規格

| 變數名稱 | 型別 | 初始值 | 說明 |
|----------|------|--------|------|
| adOpenTime | float | 0.0 | 廣告開啟後的累計秒數（timer） |
| stackCount | int | 0 | 當前漢堡層數 |

| failAdEnabled | bool | true | 失敗重試廣告開關（可由開發者設定） |
| gameState | enum | IDLE | 當前遊戲狀態機節點 |
| isFirstSession | bool | true | 是否為本次廣告展示的第一局（安全期判斷依據） |

## 7. 物理與倒塌邏輯

| 階段 | 安全期（首局） | 倒塌機率說明 |
|------|---------------|--------------|
| 第 1 – 4 層 | 完全保護（100% 安全） | 首次廣告開啟時，此範圍不觸發倒塌邏輯 |
| 第 5 層起 | 保護解除 | 倒塌機率隨層數與疊歪程度累積計算 |
| 非首局（重玩） | 無安全期 | 從第 1 層起即適用正常倒塌機率 |
| 洋蔥層 | 視安全期而定 | 洋蔥不提高倒塌率，但觸發轉盤動畫流程 |

### 7.1 倒塌計算說明

- 每層食材疊加時，系統計算漢堡整體重心偏移量
- 偏移量超過閾值時，觸發倒塌動畫並進入 FAILED 狀態
- 洋蔥層不增加基礎倒塌率，但會因轉盤結果影響倍數累計
- 閾值建議：可依測試調整，初始設定約 `stackCount * 隨機係數（0.6–1.0）`

## 8. 洋蔥轉盤邏輯

### 8.1 觸發條件

- 疊加食材時，系統隨機判定是否為「洋蔥」（建議觸發機率：20%–30%）
- 觸發後顯示全屏轉盤動畫（`gameState → ONION_SPIN`）

### 8.2 轉盤倍數區間

| 區間 | 佔比 | 倍數範圍 |
|------|------|----------|
| 一般區 | 60% | 1.1x – 1.5x |
| 良好區 | 30% | 1.5x – 2.5x |
| 爆發區 | 10% | 2.5x – 5.0x |

> 倍數採累乘方式疊加至 `currentMultiplier`

### 8.3 動畫要求

- 旋轉時間：2.0 – 3.0 秒（含緩停效果）
- 結果顯示：轉盤停止後，放大顯示倍數值 1.5 秒
- 自動關閉後恢復 STACKING 狀態

## 9. 音效規格

### 9.1 音效列表

| 音效 ID | 觸發時機 | 說明 |
|---------|----------|------|
| sfx_add | 每次疊層 | 輕快「叮」聲 |
| sfx_onion | 洋蔥轉盤啟動 | 神秘轉動聲 |
| sfx_multiplier | 倍數揭曉 | 歡呼或鼓聲 |
| sfx_serve | 出餐成功 | 收銀機聲或歡呼聲 |

### 9.2 音效控制

- 頁面頂部「音效開啟 / 關閉」切換按鈕（icon 式）
- 預設狀態：靜音（尊重手機靜音設定）
- 音效以 Base64 形式內嵌於 HTML，無外部載入

## 10. QA 開發工具規格

頁面底部提供開發者用 QA 控制面板（Dev Panel），正式上線版可隱藏或移除。

| 按鈕 / 捷徑 | 觸發動作 | 測試目的 |
|-------------|----------|----------|
| 設定為 11 秒 | `adOpenTime = 11` | 驗證「< 12 秒」出餐行為（不跳廣告） |
| 設定為 16 秒 | `adOpenTime = 16` | 驗證「> 12 秒」出餐廣告彈窗行為 |
| 設定為 26 秒 | `adOpenTime = 26` | 驗證「> 25 秒」全局攔截行為 |
| 下一層必出洋蔥 | `forceOnion = true` | 強制下一層食材為洋蔥，測試轉盤流程 |
| 快速自動追加 4 層（安全期測試） | 自動疊 4 層，`isFirstSession = true` | 測試首局前 4 層安全期保護是否正常運作 |

### 10.1 即時日誌系統

- 即時顯示系統事件（廣告觸發、狀態轉換、倒塌判定等）
- 支援「Clear」按鈕清空日誌
- 高亮標示三個廣告觸發時間點
- 開發者可切換「廣告策略追蹤」與「即時系統日誌」兩種模式

## 11. 技術規格

### 11.1 檔案限制（Google Ads Playable Ad 規範）

- 單一 HTML 檔案，所有資源（JS / CSS / 圖片 / 音效）完全內嵌
- 初始 HTML 檔案大小上限：5 MB
- 不可使用 localStorage / Cookie / 外部 CDN 資源
- 不可使用 WebGL 或需要外掛的技術

### 11.2 相容性要求

| 平台 | 最低版本 |
|------|----------|
| iOS Safari | 13+ |
| Android Chrome | 80+ |
| 桌面 Chrome | 80+（預覽用） |

### 11.3 效能目標

- 首次互動延遲（FID）：< 100ms
- 動畫幀率：60 fps（一般情況）/ 30 fps（低階裝置最低保障）
- 轉盤動畫使用 CSS Animation 或 `requestAnimationFrame` 實作
