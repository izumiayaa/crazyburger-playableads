## 2. 專案概述

### 2.1 廣告形式

- 格式：Google Ads Playable Ad（HTML5，單一自包含 index.html）
- 目標受眾：行動裝置用戶（iOS / Android）
- 主要 CTA：引導用戶前往 Google Play / App Store 下載安裝

### 2.2 遊戲玩法概述

玩家透過點擊「加料」按鈕不斷疊加漢堡食材，同時累積倍數收益。

- 每次點擊「加料」：新增一層食材（肉排、培根、生菜、番茄），並隨機獲得1x-3x倍數
- 特殊食材「雞蛋」：滑動時偽裝為普通食材外觀，落地後才揭露為 🥚 雞蛋及連續疊加層數並觸發「加蓋機會」，系統自動連續疊加 3-6 層普通食材（倍數介於 1x-3x），
- 點擊「出餐」：顯示簡化版報獎框（恭喜獲得 + 漢堡 + 實際金額，2秒後自動關閉），隨後進入 CTA 頁面
- 漢堡偏移過重：自動倒塌，顯示簡化版報獎框（2秒後自動關閉），重新開始遊戲

## 3. 廣告觸發邏輯規格

本廣告共設計三種觸發時機，開發者需依照以下規則實作：

| 觸發條件 | 類型 | 行為描述 | 備註 |
|----------|------|----------|------|
| 倒塌後重試 | 失敗驅動 | 玩家倒塌後再次點擊「加料」時，跳出商店廣告一次，關閉後可繼續遊玩 |  |
| 點擊「出餐」(階段A) | 出餐限制 | 玩家點擊「出餐」時，先顯示簡化版報獎框(2秒)，隨後直接導向 CTA 頁面（不可關閉） | 一律觸發，不限時間 |
| 開啟時間 > 25 秒 (階段B) | 全局攔截 | 玩家進行畫面的任意點擊操作（含加料按鈕），直接導向 CTA 頁面（不可關閉） | 任意點擊即觸發 |

### 3.1 CTA 頁面規格

CTA 頁面為最終導流頁面，一經進入皆不可關閉且無法回到原本遊戲流程。

| 元素 | 說明 |
|------|------|
| 遊戲 LOGO + 機台名稱 | 🍔 圖示 + 「加料蓋漢堡」文字 |
| 標語文字 | 「檯子最多，開講最瘋」 |
| 金好運 LOGO | 🎰 金好運 品牌識別 |
| 提示文字 | 「立即前往，領取你的百萬金幣！」 |
| 下載按鈕 | 「一鍵暴富 🚀」— 點擊後開啟商店頁面彈窗 |

### 3.2 簡化版報獎框規格

所有結算場景均使用同一個簡化版報獎框（統一視窗）：

| 項目 | 說明 |
|------|------|
| 標題文字 | 🎉 恭喜獲得 |
| 圖示 | 🍔 漢堡（大尺寸） |
| 金額顯示 | 💰 實際計算金額（$本金 × 累積倍數）💰 |
| 顯示時長 | 2 秒後自動關閉 |
| 使用場景 | 出餐結算（→ CTA 頁面）、倒塌結算（→ 重新開始遊戲） |

### 3.3 安全期（Safe Zone）說明

為確保用戶首次體驗廣告時能充分進行互動，系統設計「漢堡安全期」機制：

- 適用範圍：僅限本次廣告展示的「第一局」
- 保護層數：前 4 層擁有完美保護，不會觸發倒塌邏輯
- 解除時機：第 5 層起或非首局時，安全期自動失效
- 再玩一次後：`isFirstSession = false`，後續每局從第 1 層即有倒塌可能性

## 4. 遊戲狀態機 (Game State Machine)

系統以七個主要狀態組成有限狀態機（FSM），各狀態轉換規則如下：

| 狀態名稱 | 觸發條件 | 說明 |
|----------|----------|------|
| IDLE（等待開始） | 廣告首次開啟 | 顯示漢堡底座與「加料」按鈕，等待玩家互動 |
| STACKING（疊加中） | 玩家點擊「加料」 | 每次點擊新增一層食材 |
| EGG_STACK（雞蛋連蓋） | 食材落地後揭露為雞蛋 | 自動連續疊加 3~6 層普通食材，畫面顯示「加蓋機會!」Banner |
| SERVING（出餐） | 玩家點擊「出餐」 | 顯示簡化版報獎框(2秒)，隨後進入 CTA 頁面 |
| FAILED（倒塌） | 漢堡超過物理閾值倒塌 | 顯示簡化版報獎框(2秒)，自動關閉後重新開始遊戲 |
| AD_BREAK（廣告插播） | 倒塌後重試觸發 | 彈出商店廣告；關閉後恢復上一個遊戲狀態 |
| CTA（最終導流頁） | 出餐或全局攔截觸發 | 顯示 CTA 頁面，不可關閉、不可返回遊戲 |

### 4.1 狀態轉換流程

- `IDLE` → `STACKING`：玩家首次點擊「加料」
- `STACKING` → `SERVING` → `CTA`：玩家點擊「出餐」→ 報獎框(2秒) → CTA 頁面
- `STACKING` → `FAILED`：漢堡倒塌條件成立 → 報獎框(2秒) → 重新開始
- `STACKING` → `EGG_STACK`：食材落地後揭露為雞蛋，自動連續蓋 3~6 層食材（每層 150ms）
- `EGG_STACK` → `STACKING`：連蓋結束，恢復正常操作
- `FAILED` → `AD_BREAK` → `STACKING`：玩家點擊「加料」重試時觸發商店廣告一次
- `STACKING`（任意點擊）→ `CTA`：時間 > 25 秒時，任意操作直接進入 CTA 頁面
- `AD_BREAK` → [前一狀態]：玩家關閉廣告

## 5. UI 元件規格

| 元件 ID | 類型 | 顯示內容 | 互動行為 | 顯示條件 |
|---------|------|----------|----------|----------|
| btn_add | Button | 加料 🍔 | 觸發疊層邏輯；時間 > 25 秒時直接進入 CTA 頁面 | IDLE / STACKING |
| btn_serve | Button | 出餐 $NNN,NNN | 顯示報獎框(2秒)後進入 CTA 頁面 | STACKING |
| lbl_layer_count | Label | 目前層數 X 層 | 即時更新當前漢堡層數 | 常駐 |
| egg_banner | Banner | 🥚 加蓋機會! 剩餘 X 層 | 自動顯示於畫面上方，連蓋結束後隱藏 | EGG_STACK |
| msg_modal | Modal | 簡化版報獎框（恭喜獲得/漢堡/金額） | 2 秒後自動關閉 | SERVING / FAILED |
| cta_page | Overlay | CTA 導流頁面（LOGO/標語/下載按鈕） | 「一鍵暴富」按鈕開啟商店彈窗；不可關閉 | CTA |
| cta_store_popup | Modal | 商店下載頁面 | 點擊跳轉 Google Play 安裝頁 | CTA（按鈕觸發後） |
| ad_overlay | Overlay | 商店廣告（倒塌重試用） | 關閉後回到前一個遊戲狀態 | AD_BREAK |
| timer_label | Debug Label | 廣告開啟計時（秒） | 僅開發模式顯示 | Dev Mode |

### 5.1 畫面佈局說明

- 主遊戲區（中央）：漢堡疊加視覺化展示區，採手機比例垂直置中
- 右上角資訊區：目前層數 Label
- 底部操作區：「加料」按鈕、「出餐」按鈕
- CTA 橫幅（固定底部）：「立即安裝遊玩 🚀」下載連結
- Overlay 層：雞蛋加蓋 Banner、結算 Modal、廣告 Overlay（Z-index 堆疊）

### 5.2 響應式設計

- 手機預覽區：自動偵測裝置尺寸，以 375 × 667（iPhone SE）為基準
- 食材圖示：SVG 或 PNG，支援視網膜螢幕（2x / 3x）
- 字體大小：最小 12pt，確保行動裝置可讀性

## 7. 物理與倒塌邏輯

| 階段 | 安全期（首局） | 倒塌機率說明 |
|------|---------------|--------------|
| 第 1 – 4 層 | 完全保護（100% 安全） | 首次廣告開啟時，此範圍不觸發倒塌邏輯 |
| 第 5 層起 | 保護解除 | 倒塌機率隨層數與疊歪程度累積計算 |
| 非首局（重玩） | 無安全期 | 從第 1 層起即適用正常倒塌機率 |
| 雞蛋層 | 完全保護（100% 安全） | 觸發連續蓋樓層動畫流程 |

### 7.1 倒塌計算說明

- 每層食材寬度固定不變窄，與底座相同
- 每層食材疊加時，系統計算漢堡整體重心偏移量
- 偏移量超過閾值時，觸發倒塌動畫並進入 FAILED 狀態
- 雞蛋連蓋層不觸發倒塌判定
- 閾值建議：可依測試調整，初始設定約 `stackCount * 隨機係數（0.6–1.0）`

## 8. 雞蛋加蓋機會邏輯

### 8.1 觸發條件

- 疊加食材時，系統隨機判定是否為「雞蛋」（觸發機率：12%，需至少已有 2 層）
- 雞蛋在頂部滑動時偽裝為隨機普通食材外觀，玩家無法事先辨別
- 落地後才揭露為金色 🥚 黃金雞蛋樓層，並進入 `EGG_STACK` 狀態開始自動連蓋

### 8.2 連蓋規格

| 項目 | 說明 |
|------|------|
| 連蓋層數 | 隨機 3~6 層 |
| 每層倍數 | 1.0x ~ 3.0x（隨機） |
| 層寬度 | 固定與觸發層相同，不變窄 |
| 層外觀 | 隨機普通食材（牛排、生菜、番茄、培根） |
| 對齊方式 | 自動完美對齊（不需玩家操作） |
| 連蓋速度 | 每層間隔 150ms |

### 8.3 畫面表現

- 畫面上方顯示黃色 Banner：「🥚 加蓋機會!」
- Banner 右側顯示「剩餘 X 層」即時倒數
- 連蓋結束後 Banner 自動隱藏，恢復 STACKING 狀態

### 8.4 倒塌保護

- 雞蛋觸發層及自動連蓋層均不觸發倒塌判定（100% 安全）

## 10. QA 開發工具規格

頁面底部提供開發者用 QA 控制面板（Dev Panel），正式上線版可隱藏或移除。

| 按鈕 / 捷徑 | 觸發動作 | 測試目的 |
|-------------|----------|----------|
| 設定為 11 秒 | `adOpenTime = 11` | 驗證正常遊玩狀態（尚未觸發全局攔截） |
| 設定為 16 秒 | `adOpenTime = 16` | 驗證「出餐」觸發報獎框→CTA 頁面行為 |
| 設定為 26 秒 | `adOpenTime = 26` | 驗證「> 25 秒」全局攔截→CTA 頁面行為 |
| 下一層必出雞蛋 | `forceEgg = true` | 強制下一層觸發加蓋機會，測試雞蛋連蓋流程 |
| 快速自動追加 4 層（安全期測試） | 自動疊 4 層，`isFirstSession = true` | 測試首局前 4 層安全期保護是否正常運作 |
| 重置秒數 | 重置所有廣告/CTA 狀態 | 重置 CTA 狀態、計時器，回到第一次廣告頁狀態，方便反覆測試 |

### 10.1 即時日誌系統

- 即時顯示系統事件（廣告觸發、狀態轉換、倒塌判定等）
- 支援「Clear」按鈕清空日誌
- 高亮標示三個廣告觸發時間點
- 開發者可切換「廣告策略追蹤」與「即時系統日誌」兩種模式
